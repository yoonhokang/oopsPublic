# Datastore Mode 데이터 조회 문제 해결

`users` 컬렉션 조회 결과가 0건이라는 것은 **"데이터가 없거나"** 혹은 **"엉뚱한 곳을 보고 있거나"** 둘 중 하나입니다.

### 1. 가능성 A: 네임스페이스 (Namespace) 불일치
Firestore(Datastore Mode)는 데이터를 여러 '네임스페이스'로 격리할 수 있습니다.
우리가 저장할 때 네임스페이스를 지정했는지 확인해야 합니다. (현재 코드는 지정 안 함 = `[default]`)

콘솔 상단 쿼리 입력창 근처나, **"More query options"** 같은 메뉴에서 **Namespace** 설정이 `[default]`로 되어 있는지 확인해 주세요.

### 2. 가능성 B: 데이터베이스 ID 불일치
우리가 코드를 수정하면서 데이터베이스 ID를 `(default)`로 사용하고 있는지, 아니면 다른 이름(`oopsPublic-db` 등)을 사용하고 있는지 확인이 필요합니다.

*   `js/api-config.js` 파일에서 `endpoints.firestore` URL이 정확히 어느 프로젝트/DB를 가리키는지 확인합니다.
*   콘솔 URL 상의 데이터베이스 이름과 코드의 타겟이 일치하는지 확인해야 합니다.

### 3. 가능성 C: 저장 실패 (가장 유력)
만약 콘솔의 **Entities** 탭에서 아무리 찾아도 데이터가 없다면, 실제로 저장이 안 된 것입니다.
앱에서 "Saved successfully" 메시지가 떴더라도, **다른 프로젝트**나 **다른 데이터베이스**로 저장되었을 수 있습니다.

**확인할 것:**
1.  앱 실행 후 `F12` > `Network` 탭을 켭니다.
2.  저장 버튼을 누릅니다.
3.  `commit` 또는 `runQuery` 요청의 **Response**를 확인합니다.
4.  만약 Response에 에러 없이 정상 응답(200 OK)이 왔다면, 그 응답 본문에 **"어디에 저장했는지"** 경로가 나옵니다.
    *   예: `projects/oopsPublic/databases/(default)/documents/...`

이 경로가 현재 보고 계신 콘솔의 경로와 일치하는지 확인해야 합니다.
